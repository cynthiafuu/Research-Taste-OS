# Research Taste OS Notion Workflow

## Daily Operation

Run commands from:

```bash
/Users/sylviafu/APP I created/research workflow-p0
```

Recommended commands:

```bash
.venv/bin/research-os run-pdf "/full/path/to/paper.pdf"
.venv/bin/research-os run-folder "/Users/sylviafu/Library/Mobile Documents/com~apple~CloudDocs/1_Academia/Literature" --limit 3
.venv/bin/research-os ux-v2
```

Do not ask the user to manually fill title, year, field, or journal unless auto-detection is clearly wrong. `run-pdf` should infer basic metadata and default gracefully.

## Notion UX Contract

The main page is `Research Taste OS - Simple`.

It should contain:

- A short English guide.
- A lightweight Paper Bank linked view.
- A Paper Page UX v2 guide.

Each generated paper page should use visually distinct sections:

- Blue: paper facts and Paper Card.
- Yellow or orange: Taste Memo and referee concerns.
- Green: idea extensions and promoted ideas.
- Red: risks, fatal flaws, or pipeline stops.
- Purple: My Judgment.

The `My Judgment` section should ask the user to reflect on surprise, disagreement, reusable design choices, whether to build on the paper, and an advisor question.

## English-Only Rule

All Notion content generated by this workflow should be in English, including dashboard guide text, Paper Card, Taste Memo, idea extensions, scorecards, Mini Proposal, Referee Simulation, Advisor Memo, and My Judgment prompts.

## Common Troubleshooting

- If Notion content looks cluttered, run `.venv/bin/research-os ux-v2`.
- If the Paper Bank row exists but analysis is missing, check the terminal for OpenAI quota or billing errors, then rerun `run-pdf`.
- If the linked database block appears as `Untitled` in low-level API output, do not chase that block title; Notion does not allow direct `child_database` title patching through the REST endpoint. Keep a clear `Paper Bank` heading above the view.
- If the user asks to process a whole folder, use `run-folder` with a conservative `--limit` unless they specify otherwise.
- If the user asks to keep everything in one place, avoid `--relational`.
